# ExcelMiner 业务逻辑详解

> 本文档基于 VBA 原型脚本（`业务原型/*.bas`）和提示词模板（`业务原型/*.md`、`resources/prompts/*.md`）完整描述数据从 Excel 源文件到最终 AI 分析文案的全链路业务处理逻辑。所有行号、列号、公式、格式、常量名均来自 VBA 源码。

## 零、系统架构概览

```
┌─────────────────────────────────────────────────────────┐
│                    Excel 源文件                          │
│  活动量/{公司}.xlsx（9家） + 经营报表/{公司}.xlsx（9家）│
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐

└───────────────┬─────────────────────────────────────────┘

    ┌───────────┴───────────┐

│ 无质量评分    │   │ 5维评分≥8分 + 最多2次重试 │
└───────┬───────┘   └───────────┬───────────────┘
        │                       │
        └───────────┬───────────┘
                    ▼
        ┌───────────────────────┐
        │   DeepSeek API        │
        │   deepseek-chat       │
        │   temperature=0.3     │
        │   HTTP重试: 指数退避  │
        │   不可重试: 401-403   │
        └───────────────────────┘
                    ▼

        │  分析文案 → Excel     │
        │  可直接粘贴到PPT半页  │
        └───────────────────────┘
```

**数据流关键特征**：
- 数据汇总层 **只做数值提取和累加**，不涉及 AI 调用
- 经营报表汇总 **纯值复制**（`xlPasteValues`），不做任何计算
- AI 分析层分两个阶段：先板块（3段）、后公司（9段），合计最多 12 段文案
- 板块分析跳过质量检查；公司分析有 5 维 10 分质量评分 + 最多 2 次重试

## 二、数据清洗规则

### 2.1 数字提取（ParseNumeric）

`汇总公共模块.bas` 的 `ParseNumeric` 函数（Lines 93-121）处理优先级：

| 优先级 | 输入示例    | 处理逻辑                                                    | 结果    |
| ------ | ----------- | ----------------------------------------------------------- | ------- |
| 1      | 空单元格    | `IsEmpty(v)` → 0                                            | 0       |
| 2      | 纯数值      | `IsNumeric(v)` → `CDbl(v)` 直接取值                         | 12.5    |
| 3      | 文本含数字  | 逐字符扫描，收集数字(0-9)和第一个小数点，构建数字串后 CDbl  | 见下文  |
| 4      | 纯中文/其他 | 无数字→返回 0                                                | 0       |

**VBA 逐字符扫描逻辑**：

```
s = CStr(v)
numStr = ""
For i = 1 To Len(s)
    If IsNumeric(Mid$(s,i,1)) Or Mid$(s,i,1) = "." Then
        numStr = numStr & Mid$(s,i,1)
    End If
Next i
If Len(numStr) > 0 And IsNumeric(numStr) Then ParseNumeric = CDbl(numStr) Else ParseNumeric = 0
```

**注意事项**：
- 多小数点时仅第一个有效（后续小数点也被拼入，但 `IsNumeric` 可能拒绝对 `"12.5.3"` 这样的串返回 True，故结果为 0）
- 不处理百分号（`%` 非数字非小数点，被跳过）
- 不处理货币符号（`¥`、`$` 等直接跳过）
- 不处理负号（`-` 被跳过，负值会变正值）
- `#N/A` 等错误值通过 `IsEmpty`/`IsNumeric` 检查后返回 0

### 2.2 数字提取增强版（ExtractNumberFromCell）

`酒店数据汇总.bas` 的 `ExtractNumberFromCell` 函数（Lines 175-216）专为酒店业态设计，新增了加法表达式和错误值支持：

| 优先级 | 输入示例      | 处理逻辑                                                | 结果  |
| ------ | ------------- | ------------------------------------------------------- | ----- |
| 1      | 空值          | `IsEmpty` → 0                                           | 0     |
| 2      | 错误值        | `IsError` → 0（ParseNumeric 无此分支）                  | 0     |
| 3      | 纯数值        | `IsNumeric` → `CDbl`                                    | 12.5  |
| 4      | 含 `+` 的文本 | 按 `+` 分割（`Split(strVal, "+")`），每段调 `ExtractSingleNumber` 求和 | 1101  |
| 5      | 普通文本      | 调 `ExtractSingleNumber` 提取首个连续数字串              | 1736  |

**ExtractSingleNumber 实现**（Lines 219-250）：

```
逐字符扫描:
  数字(0-9) → 拼接到 numStr，foundDigit=True
  "." 且 foundDigit 且 !foundDot → 拼接到 numStr，foundDot=True
  已找到数字后遇到非数字非小数点 → Exit For（终止扫描）

与 ParseNumeric 的关键区别：找到数字后遇到非数字字符即停止，
不会继续向后扫描，因此 "直播1736场" 只提取 "1736"，
而 ParseNumeric 会提取所有数字和点号，可能产生意外组合。
```

示例：
- `"直播1736场"` → 跳过"直播" → 扫描 `1736` → 遇"场"时 `foundDigit=True` 且 ch≠数字/小数点 → Exit → 提取 `1736`
- `"3.5万"` → `3.5` → 遇"万"终止 → 提取 `3.5`
- `"1+1000"` → 分割为 `["1", "1000"]` → `1 + 1000 = 1001`
- `"85%"` → `85` → 遇 `%` 终止 → 提取 `85`（不处理百分号）

#### 3.1.2 列号规律（与商写共用，酒店不同）

达成列隔列排列（计划列 | 达成列），第 m 月达成列号 = `2×m + 2`：
- 1月达成 = D列（列号4）
- 2月达成 = F列（列号6）
- 3月达成 = H列（列号8）
- ...12月达成 = Z列（列号26）

#### 3.1.3 计算公式

**YTD 累加**：使用 `汇总公共模块.SumAchievementCols`（`col = 2×m + 2`，`m=1..numMonths`）

**人力派生指标**（VBA Lines 160-183）：
| 指标     | 公式                                                  | 计算方式                             |
| -------- | ----------------------------------------------------- | ------------------------------------ |
| 期初人力 | 1月达成值                                             | `SafeRead(ws, ROW_BASE_HR, 4)`       |
| 当月净增 | `YTD入职 − YTD离职`                                   | `netAdd = ytdHRIn - ytdHROut`        |
| 月末人力 | `期初人力 + YTD入职 − YTD离职`                         | `endHR = baseHR + ytdHRIn - ytdHROut`|
| 平均人力 | `Σ(各月月末人力) ÷ 月数`                               | 逐月累加计算，非一次性除法            |

平均人力的逐月计算逻辑（VBA Lines 174-183）：
```vba
cumIn = 0 : cumOut = 0 : monthlyEndHR = 0
For m = 1 To numMonths
    col = 2 * m + 2
    cumIn = cumIn + SafeRead(wsSource, ROW_HR_IN, col)
    cumOut = cumOut + SafeRead(wsSource, ROW_HR_OUT, col)
    monthlyEndHR = monthlyEndHR + (baseHR + cumIn - cumOut)
Next m
avgHR = monthlyEndHR / numMonths
```

**效率派生指标**（由 Excel 公式计算）：
| 指标     | 公式                                             | 写入方式          |
| -------- | ------------------------------------------------ | ----------------- |
| 活动率   | `=C8/C7`（开单人数 ÷ 平均人力）                  | `WriteFormulas` 写入公式 |
| 件均保费 | `=C10/C16`（承保新单 ÷ 件数）                    | 同上              |
| 人均保费 | `=C10/C6`（承保新单 ÷ 月末人力）                 | 同上              |

#### 5.3.2 保险业态数据过滤

保险业态分析的数据构建（`BuildInsuranceDataPrompt` 函数，Lines 167-189）有特殊处理：

1. 扫描 A 列找到 `"规模保费"` 标记，进入规模保费区段
2. 对规模保费区段的每一行，调用 `ParseMonthLabel` 提取月份数字（如 `"1月"` → 1）
3. 若 `monthNum > currentMonth`，跳过该行（`GoTo NextRow`）
4. 这确保 AI 不会看到 `#N/A` 对应的月份行

`ParseMonthLabel` 函数（Lines 204-210）：
```vba
t = Trim$(label)
If Right$(t, 1) = "月" Then t = Left$(t, Len(t) - 1)
If IsNumeric(t) Then ParseMonthLabel = CLng(t) Else ParseMonthLabel = 0
```

#### 5.4.5 重试机制

两层重试设计（`AnalyzeWithQualityCheck` 函数，Lines 235-274）：

| 层级     | 配置                        | 实现                                                        |
| -------- | --------------------------- | ----------------------------------------------------------- |
| HTTP 层  | 1 次（调用时传 `maxRetries=1`） | `CallDeepSeekAPI(...)` 内部处理，指数退避                    |
| 分析层   | 最多 2 次（`MAX_RETRIES=2`）   | 质量评分 < 8 → 重新调用 API                                  |
| 重试间隔 | 2秒（`RETRY_DELAY_MS=2000`）   | `Application.Wait Now + TimeSerial(0,0,2)`                   |
| 通过阈值 | **≥ 8 分**                  | `quality.Score >= 8`                                        |
| 未达标标记 | —                         | 追加 `[质量提示：本分析质量评分 X/10，部分指标描述可能不完整]` |

关键设计决策：**调用 DeepSeekAPI 时将 `maxRetries` 传为 1**（Line 243），避免 HTTP 层 3 次 × 分析层 3 次 = 9 次嵌套重试的指数爆炸。

## 六、API 调用公共模块详解

**VBA 源码**：[DeepSeekAPI.bas](DeepSeekAPI.bas)

### 6.1 HTTP 客户端

`CreateHTTPObject` 函数（Lines 141-146）：
```vba
On Error Resume Next
Set CreateHTTPObject = CreateObject("MSXML2.ServerXMLHTTP.6.0")
If Err.Number <> 0 Then
    Err.Clear
    Set CreateHTTPObject = CreateObject("MSXML2.XMLHTTP")
End If
On Error GoTo 0
```

优先使用 `ServerXMLHTTP.6.0`（支持 `setTimeouts`），失败回退 `XMLHTTP`（不支持超时设置）。

### 6.2 JSON 构建

`BuildJSONRequest` 函数（Lines 148-150）：
```json
{
  "model": "deepseek-chat",
  "messages": [
    {"role": "system", "content": "<system prompt>"},
    {"role": "user", "content": "<user prompt>"}
  ],
  "temperature": 0.3,
  "max_tokens": 1000
}
```

JSON 在 VBA 中手动拼接，通过 `EscapeJSONString` 转义特殊字符。

### 6.3 JSON 字符串转义

`EscapeJSONString` 函数（Lines 90-108）处理的字符：

| 字符          | 转义为   |
| ------------- | -------- |
| `\`           | `\\`     |
| `"`           | `\"`     |
| `vbCr` (0x0D) | `\r`     |
| `vbLf` (0x0A) | `\n`     |
| `vbTab`       | `\t`     |
| `vbBack`      | `\b`     |
| `vbFormFeed`  | `\f`     |

使用预分配缓冲区（`Space$(Len(text)*2)`）提升拼接效率。

### 6.4 响应解析

`ParseAPIResponseSimple` 函数（Lines 110-135）提取 `content` 字段：

1. 定位 `"content":"` 位置（`CONTENT_PREFIX_LEN=11`）
2. 从该位置扫描，追踪转义 `\` 状态
3. 遇到未转义的 `"` 终止
4. 反转义：`\"→"`, `\n→vbCrLf`, `\t→vbTab`, `\r→""（去除）`, `\/→/`, `\b→vbBack`, `\f→vbFormFeed`, `\\→\`

> **注意**：`\\` 替换在最后执行，确保不会把已反转义的 `\` 再次消费。`\r` 直接去除而非保留为 `vbCr`，因为 Windows 换行用 `vbCrLf`，保留 `\r` 会产生多余回车符。

### 6.5 HTTP 重试逻辑

`CallDeepSeekAPI` 函数（Lines 152-232）的 HTTP 层重试：

- 最多 `maxRetries` 次（默认 3 次，调用方可覆盖）
- 指数退避：`waitSec = 2^retryCount`（1s, 2s, 4s, ...）
- 实现方式：`Application.Wait (Now + TimeValue("0:00:" & CStr(waitSec)))`
- **不可重试状态码**（`IsRetryableStatus`，Lines 234-241）：401, 402, 403, 422
- 其他状态码和超时均为可重试
- 支持 `ServerXMLHTTP` 的超时设置：`setTimeouts timeoutMs, timeoutMs, timeoutMs, timeoutMs`（resolve, connect, send, receive 统一值）

### 6.6 API Key 管理

`GetApiKey` 函数（Lines 42-84）：
- 文件路径：`Environ("USERPROFILE") & "\." & API_KEY_FILE` → `%USERPROFILE%\.dskey`
- 格式：`[EXCEL]=sk-xxxxxxxx`
- 逐行扫描，查找 `EXCEL=` 前缀（`vbTextCompare` 大小写不敏感）
- 跳过 `#` 开头的注释行
- 支持 `debugMode` 参数输出错误信息到 VBE 立即窗口

### 6.7 数据提示词构建

`BuildDataPrompt(dataArray)` 函数（Lines 247-260）将 VBA 二维数组转换为管道分隔的表格文本：
```
数据表格如下：
| 指标1 | 值1 | 值2 | ... |
| 指标2 | 值3 | 值4 | ... |
```
空单元格输出空 `|` 段（`| |`），非空单元格用空格包裹（`| 值 |`）。

### 6.8 结果处理

**`CleanAnalysisResult`**（Lines 266-277）：去除空行
```
逐行 Trim → 非空才保留 → 重新 Join
```

**`WriteAnalysisResult`**（Lines 279-288）：写入单元格并设置格式
```vba
.Value = result
.WrapText = True
.HorizontalAlignment = xlLeft
.VerticalAlignment = xlTop
```

### 6.9 调试模式

`DEBUG_MODE` 开关（Line 10）控制：
- `CallDeepSeekAPI` 的请求/响应日志（长度、状态码、错误详情）
- `GetApiKey` 的配置错误信息
- `AnalyzeCompany` 的进度和质量评分日志
- `WriteAnalysisResult` 的写入确认日志

所有日志通过 `FormatLog`/`FormatError` 统一格式化，输出到 VBE 立即窗口（`Debug.Print`）。
